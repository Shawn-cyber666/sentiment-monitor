# 🎯 Vivo X300U 舆论监测网站 - 完整项目交付清单

## 📦 项目概况

**项目名称:** Vivo X300U 舆论监测实时展示系统  
**项目类型:** 企业级舆论监测和用户反馈分析平台  
**技术栈:** React + Node.js + Python + MongoDB + Redis  
**部署方式:** Docker + Docker Compose

---

## 📋 交付文件清单

### 🎨 前端相关

| 文件名 | 类型 | 说明 | 用途 |
|--------|------|------|------|
| `sentiment-monitor.jsx` | React 组件 | 完整的舆论监测仪表板 | 直接用于 React 项目 |

**功能特性:**
- ✅ 实时数据展示（3秒更新一次）
- ✅ 多平台 KPI 统计（小红书、微博、B站）
- ✅ 关键词云展示（按频率和情感排序）
- ✅ 24小时趋势图表
- ✅ 情感分布饼图
- ✅ 代表评论展示
- ✅ 专业的深色主题设计

**使用方式:**
```jsx
import SentimentMonitor from './sentiment-monitor';

export default function App() {
  return <SentimentMonitor />;
}
```

### ⚙️ 后端相关

| 文件名 | 类型 | 说明 |
|--------|------|------|
| `backend-server.js` | Node.js | Express 服务器，包含完整 API 和 WebSocket |
| `package.json` | 配置 | 后端依赖配置 |
| `backend-Dockerfile` | Docker | 后端容器化配置 |

**API 端点:**
```
GET  /api/dashboard                  # 获取仪表板数据
GET  /api/comments/keyword/:keyword  # 按关键词查询评论
GET  /api/sentiment-distribution     # 获取情感分布
GET  /api/trends                     # 获取趋势分析
POST /api/comments                   # 添加新评论
WS   /socket.io                      # WebSocket 实时推送
```

## 🚀 快速开始 (3 种方式)

### 方式 1: Docker 快速启动 (推荐) - 5分钟

```bash
# 1. 下载项目文件并进入目录
cd sentiment-monitor

# 2. 复制并编辑环境配置
cp .env.example .env
# 编辑 .env 文件，填入必要的 API 密钥

# 3. 启动所有服务
docker-compose up -d

# 4. 打开浏览器
# 前端: http://localhost:3000
# 后端: http://localhost:5000
# 监控: docker-compose logs -f
```

### 方式 2: 本地开发环境 - 15分钟

```bash
# 前端
cd frontend
npm install
npm start  # http://localhost:3000

# 后端 (新终端)
cd backend
npm install
npm run dev  # http://localhost:5000

# 爬虫 (新终端)
cd crawler
pip install -r requirements.txt
python crawler_spider.py
```

### 方式 3: 云平台部署

参考 `舆论监测网站_实现指南.md` 中的部署章节

## 🎯 核心功能详解

### 1. 实时数据可视化

**功能:** 展示来自三个平台的实时评论统计

```
┌─────────────────────────────────────┐
│     平台         评论数    情感评分   │
├─────────────────────────────────────┤
│  小红书        2,847        72      │
│  微博          3,621        68      │
│  B站           1,892        75      │
└─────────────────────────────────────┘
```

### 2. 关键词监控

**功能:** 自动提取和统计高频关键词

```
关键词        频率   情感  趋势
────────────────────────────
屏幕素质      342    85   ↑
拍照能力      298    79   ↑
续航能力      267    62   ↓
散热控制      245    58   ↓
```

### 3. 情感分析

**功能:** 对每条评论进行情感打分（0-100）

- 非常积极 (85-100) - 绿色
- 积极 (70-84) - 浅绿
- 中立 (55-69) - 灰色
- 消极 (35-54) - 橙色
- 非常消极 (0-34) - 红色

### 4. 趋势分析

**功能:** 显示 24 小时内的评论数量变化

```
时间  小红书  微博  B站
────────────────────
00:00   12    18    8
04:00    8    12    6
08:00   24    31   18
12:00   42    58   32
16:00   38    52   28
20:00   45    62   35
23:00   28    38   20
```

### 5. 代表性评论

**功能:** 展示高赞、代表性的用户评论

## 📊 数据流向

```
用户评论 (小红书、微博、B站)
    ↓
爬虫采集 (3 个专业爬虫)
    ↓
数据处理 (分词、情感分析、关键词提取)
    ↓
存储 (MongoDB 主库 + Redis 缓存)
    ↓
API 服务 (Express 后端)
    ↓
前端展示 (React 组件)
    ↓
实时推送 (WebSocket)
```

## 🔧 配置说明

### 必需配置

编辑 `.env` 文件，填入以下信息：

```ini
# 数据库
MONGODB_URI=mongodb://localhost:27017/sentiment-monitor
REDIS_HOST=localhost
REDIS_PORT=6379

# 小红书 (可选 - 使用模拟数据)
XIAOHONGSHU_API_KEY=your_key

# 微博 (可选 - 使用模拟数据)
WEIBO_ACCESS_TOKEN=your_token

# B站 (可选 - 使用模拟数据)
BILIBILI_SESSDATA=your_sessdata

# 服务器
PORT=5000
NODE_ENV=development
```

### 默认设置

如果不配置外部 API，系统会使用**模拟数据**演示，即开即用。

## 🐛 故障排除快速参考

| 问题 | 解决方案 |
|------|---------|
| MongoDB 连接失败 | `mongosh mongodb://localhost:27017` |
| Redis 连接失败 | `redis-cli ping` |
| 前端白屏 | 检查后端是否运行，查看浏览器控制台错误 |
| 爬虫不工作 | 检查 API 密钥配置，查看爬虫日志 |
